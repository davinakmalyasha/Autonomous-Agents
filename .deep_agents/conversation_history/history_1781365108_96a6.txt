## Planning

### Goal
Implement an in-memory key-value store (`kv_store.py`) with:
- `get(key)` – returns value or `None` if missing/expired.
- `set(key, value, ttl=None)` – stores value with optional TTL in seconds.
- `delete(key)` – removes key, raises `KeyError` if missing.
- `backup_to_file(path)` – saves state as JSON (including expiry timestamps).
- `restore_from_file(path)` – loads state from JSON (preserving expiry).
- TTL expiration: keys become inaccessible after TTL passes.
- Time dependency must be mockable for testing.

### Proposed Changes
- Create `kv_store.py` with class `KVStore`.
- Use injectable `time_func` (default `time.time`) for all time references.
- Internal storage: `{key: (value, expiry)}` where `expiry` is `None` (no TTL) or absolute timestamp.
- `backup_to_file` serializes: `{key: {"value": value, "expires_at": expiry}}`.
- `restore_from_file` loads that format.
- Create `test_kv_store.py` with pytest tests covering:
  - Basic get/set/delete.
  - TTL expiration (using a mock clock).
  - Backup and restore of empty and non-empty store.
  - Edge cases: delete missing, get expired returns `None`.

### Steps
1. Write `planning.md`.
2. Implement `kv_store.py`.
3. Implement `test_kv_store.py`.
4. Run tests and fix any failures.
